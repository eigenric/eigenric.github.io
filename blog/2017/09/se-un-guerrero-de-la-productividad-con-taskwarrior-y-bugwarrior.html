Sé un guerrero de la productividad con Taskwarrior y Bugwarrior

1 sep 2017

Taskwarrior es una herramienta de línea de comandos para la gestión de tareas, diseñada para ser potente, flexible y personalizable. Ideal para usuarios que prefieren trabajar en terminal, permite capturar, rastrear y organizar tareas de manera eficiente.

Ejemplos de Uso

  • Añadir una tarea:
$ task add "Leer el capítulo 1 del libro X"
  • Añadir una tarea con proyecto:
$ task add "Escribir introducción del blog" project:blog
  • Listar tareas:
$ task list

Listar tareas de un proyecto:

$ task project:blog list
  • Completar una tarea:
$ task 1 done
  • Modificar una tarea:
$ task 1 modify priority:H

(También se puede usar task x mod)

  • Añadir una tarea con fecha de vencimiento y etiqueta:
$ task add "Comprar leche" due:tomorrow +compras
  • Filtrar tareas por etiqueta:
task +compras list

Por otro lado, Bugwarrior es una herramienta excelente para gestionar issues de GitHub y otros servicios como Trello. Cuenta con una documentación completa y está escrito en Python, lo que facilita la contribución al proyecto.

Instalación

Para instalar Bugwarrior y las dependencias necesarias, usa el siguiente comando:

$ pip install bugwarrior[keyring]

Esto incluye las dependencias necesarias para usar bugwarrior-vault, la herramienta que gestiona las claves.

Crea un archivo .config/bugwarriorrc con el siguiente contenido:

[general]
targets = pwaqo_github_account

[pwaqo_github_account]
service = github
github.login = eigenric
github.username = eigenric
github.password = @oracle:use_keyring
github.description_template = {{githubnumber}}:{{githubtitle}}

El campo github.password = @oracle:use_keyring indica que la clave se almacenará utilizando bugwarrior_vault

Almacenamiento de Claves Registra la contraseña en el anillo de claves con el siguiente comando:

shell Copiar código bugwarrior-vault set - github://eigenric@github.com/eigenric eigenric Introduce la contraseña cuando se te solicite.

Filtros Personalizados Es posible configurar Bugwarrior para obtener únicamente las issues que cumplan ciertos requisitos, como:

Asignación propia Etiquetas específicas Proyectos específicos Ejecución de la Sincronización Para ejecutar la sincronización y obtener las issues, utiliza el siguiente comando:

shell Copiar código $ bugwarrior-pull Esta configuración te permitirá gestionar tus issues de GitHub de manera eficiente y personalizada, aprovechando la flexibilidad y potencia de Bugwarrior.
