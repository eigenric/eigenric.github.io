Rvalue en C++11

19 nov 2018

Nos encontramos con el siguiente error referido al fragmento de código:

template <class T>
T& Stackl<T>::pop() {
    return lista.pop_front();
}

./include/stackl.cpp:6:28: error: invalid
initialization of non-const reference of type
‘int&’ from an rvalue of type ‘void’

Al parecer lista.pop_front() devuelve rvalue, y por tanto no se puede inicializar una referencia no constante (int&), rvalue de tipo void.

Al parecer:

"Every expression is either an lvalue or an rvalue."

Lvalues: Objetos que persisten después de una expresión, obj, *ptr, ptr[index], ++x.

Rvalues: Temporales, mueren al final tras ; 1729, x+y, std::string("now"), x++

Para usar &, el operador de dirección, es necesario que sea un lvalue. Si no, es un rvalue.

C++11 introduce el rvalue reference (&&).
