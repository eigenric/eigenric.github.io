Pseudoaleatorio

9 oct 2021

Es conocido que los ordenadores no pueden producir verdaderos número aleatorios. Es por ello que se habla de generadores de números pseudoaleatorios.

Aprendiendo Rust para resolver el Advent of Code 2020 con programación funcional.

Me he encontrado en la situación de generar una lista números tal que sólo dos de ellos sumen 2020.

Itertools combinations nos da una primera solución sencilla para datos pequeños. Itertools está disponible tanto en Python como en Rust.

Curiosamente, he generado una lista de 10_000 valores entre 0 a 20_000, gracias a randint y sólo dos de ellos suman 2020.

¿Cual era la probabilidad de que pasara esto?

Investigando, randint sigue una distribución uniforme (discreta). Es decir la P(X=x) del rango es 1/20_000.

Ahora, como calculamos la probabilidad de que la suma de dos números tenga cierto valor?

Conjeturamos que la variable aleatoria suma X+Y de dos uniformes es la que mide tal caracteristica.

Es la uniforme reproductiva? Es decir, cual es PMF de una suma de uniformes?

P(X=x) = 1/n

P(Y=y) = 1/n

P(X+Y=z) = 1/(b-a)

2012 + 8 -> 4.04e-5 1020 + 1000 -> 4.04e-2

De 10_000 números entre 0 y 200_000 uniformemente distribuidos. Es decir que P(X=x) = 1/2e5,

cúal es la probabilidad de que exactamente una pareja sume 2020?

Cuantas parejas hay? 10_000C2 = 50M Cuantas parejas suman 2020?

0,2020 1,2019 ... 1009,1011

1010 posibles parejas que suman 2020!

La probabilidad de que salgan una pareja particular generados dos de forma uniforme en rango 0, 2e6

1/200_000^2 = 1/4e10 = 2.5e-11

Luego este es la probabilidad de exito

B(50M, 2.5e-100)

P(Y=y) = 2.5e-11 * 0.999999999975**9_999

= 2.49e-11

E[Y] = 5e7 * 2.49e-11 = 0.001
